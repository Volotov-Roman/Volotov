# branches
different 'timelines' of your project. you can create new ones, switch between them and merge them back together
проверить статус репозитория: git status

подготовить новый файл для комита: git add branch-how-to.md

проверить статус репозитория: git status

сделать комит: git commit -m “branch instructions”

отправить изменения на сервер: git push
